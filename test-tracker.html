ClickGuard Tracker Test Page

⚠️ Important: This page includes TWO tracker scripts for testing dynamic endpoint detection. Make sure to replace "your-live-domain.com" with your actual live domain before testing.

Script Sources:

  • Local Development: http://localhost:3001/api/v1/tracker/script
  • Live Domain: https://your-live-domain.com/api/v1/tracker/script (replace with actual domain)

The tracker will automatically detect which script source loaded it and use that domain for data submission.

Note: This enhanced tracker automatically collects comprehensive device fingerprinting data including canvas fingerprint, screen resolution, timezone, and language settings.

Enhanced Features Implemented:

  • Canvas Fingerprinting: Unique device identification via canvas rendering
  • Screen Resolution: Captures window.screen.width and height
  • Timezone Detection: Uses Intl.DateTimeFormat().resolvedOptions().timeZone
  • Language Detection: Captures navigator.language
  • Dynamic Endpoint Detection: Automatically determines backend URL from script src
  • Enhanced Error Handling: Improved retry logic with exponential backoff
  • IP Enrichment: Backend enriches data with geolocation via ip-api.com
  • Fraud Detection: Rule engine analyzes clicks for suspicious patterns

How to Use

This page demonstrates how to integrate the enhanced ClickGuard tracking script into your website with multiple domains.

1. Include the Scripts

<!-- Local development -->
<script src="http://localhost:3001/api/v1/tracker/script"></script>

<!-- Live domain -->
<script src="https://your-live-domain.com/api/v1/tracker/script"></script>

2. Dynamic Endpoint Detection

The tracker automatically detects which script source loaded it and uses that domain for data submission. This allows seamless switching between development and production environments.

3. Optional: Get Session ID

const sessionId = ClickGuard.getSessionId();
